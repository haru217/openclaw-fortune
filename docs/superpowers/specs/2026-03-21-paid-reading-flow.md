# 個別鑑定フロー設計書

## 概要

LINE トーク内で個別鑑定を受付→鑑定生成→PDF配信するフロー。決済は後から Stripe を挟める設計にしておく。

## 設計の前提

パーソナライズ感を決める要素は3つ。

1. **入力の解像度** — ユーザーの状況をどれだけ正確に把握できるか
2. **出力の具体性** — 日付・人物・行動レベルで具体的に書けるか
3. **相談者の言葉がPDFに反映されている実感** — 最重要。占いの精度より「自分の状況を分かった上で答えてくれている」と感じるかが2,000円の満足度を決める

## ユーザーフロー

```
① サービス紹介カード +「受ける」ボタン
② 基本情報入力（呼び名・出生時刻）
③ 大項目4つから選択
④ サブメニュー3つから選択
⑤ 構造化質問（選択2問 + 自由記述1問）
⑥ 確認画面 → 受付完了 → 2〜3時間後にPDF送信
```

### ① サービス紹介カード

- カイのキャラで語る
- 何が得られるかを具体的に記載（西洋占星術×数秘術×タロットの3軸鑑定、PDF3ページ）
- 所要時間「鑑定結果は数時間後にお届け」
- 価格は記載しない（後で Stripe 挟む時に追加）
- 「受ける」ボタン（action: message, text: "鑑定を受ける"）

### ② 基本情報入力

生年月日・星座は登録済みのため不要。追加で取得するもの：

- **呼び名（必須）** — PDF内で「あなた」ではなく名前で呼ぶ。パーソナライズ感が大幅に上がる。LINE表示名は絵文字やニックネームで使えないため手入力
- **出生時刻（任意）** — アセンダント算出用。「分からない」選択肢付き。「より精密な鑑定になります」と添えて入力率を上げる。母子手帳に書いてある人も多い
- **性別は取らない** — 入力ハードルになるし、今の時代センシティブ。プロンプト側で性別に依存しない文面を生成する

### ③ 大項目選択（4つ）

| ID | 大項目 |
|----|--------|
| general | 総合鑑定 |
| love | 恋愛・人間関係 |
| career | 仕事・キャリア |
| destiny | 運命の転機 |

### ④ サブメニュー選択（各3つ）

| 大項目 | サブメニュー |
|--------|-------------|
| 総合鑑定 | 今後の運勢の流れ / 人生の方向性 / 今の悩みへのアドバイス |
| 恋愛・人間関係 | 片思い・出会い / パートナーとの関係 / 復縁・別れ |
| 仕事・キャリア | 転職・適職 / 今の仕事の行方 / 人間関係（職場） |
| 運命の転機 | 転機の時期 / 大きな決断の指針 / 人生の使命・テーマ |

### ⑤ 構造化質問

サブメニュー選択後に、サブメニュー別の質問セットを表示する。構成は全パターン共通で「状況 × 関心事 × 自由記述」の3点セット。

- **Q1: 状況選択**（単一選択）— 現在の状態を把握
- **Q2: 関心事選択**（複数選択可）— 何を知りたいか
- **Q3: 自由記述**（任意）— プレースホルダーに具体例を表示

Q1・Q2で最低限の文脈が構造的に取れるため、Q3が空でもパーソナライズの下限が保証される。Q3が埋まればさらに深くなる。

#### 質問セット一覧

**仕事・キャリア → 転職・適職**
- Q1: 現在の状況（転職活動中 / 内定済み・入社前 / 転職を検討中 / 今の仕事に不満がある）
- Q2: 知りたいこと（タイミング / 向いている職種・業界 / 人間関係 / 成功のポイント / 注意すべきこと）
- Q3: 自由記述（例: 「4月から新しい会社に入社します。期待もあるけど不安もあって…」）

**仕事・キャリア → 今の仕事の行方**
- Q1: 現在の状況（順調 / 停滞感がある / 異動・配置転換があった / 評価に不満がある）
- Q2: 知りたいこと（今後の展望 / 昇進・昇給 / スキルアップの方向 / 辞め時 / 注意すべきこと）
- Q3: 自由記述

**仕事・キャリア → 人間関係（職場）**
- Q1: 現在の状況（上司との関係 / 同僚との関係 / 部下との関係 / チーム全体の雰囲気）
- Q2: 知りたいこと（改善のヒント / 味方になる人 / 距離を置くべき人 / コミュニケーション方法 / 転機の時期）
- Q3: 自由記述

**恋愛・人間関係 → 片思い・出会い**
- Q1: 現在の状況（気になる人がいる / 出会いがない / マッチングアプリ利用中 / 告白を迷っている）
- Q2: 知りたいこと（相手の気持ち / 出会いの時期 / アプローチ方法 / 相性）
- Q3: 自由記述

**恋愛・人間関係 → パートナーとの関係**
- Q1: 現在の状況（安定している / マンネリを感じる / すれ違いが増えた / 結婚を考えている）
- Q2: 知りたいこと（関係の行方 / コミュニケーション改善 / 結婚のタイミング / 相性の深堀り）
- Q3: 自由記述

**恋愛・人間関係 → 復縁・別れ**
- Q1: 現在の状況（別れたばかり / 復縁を迷っている / 別れを検討中 / 連絡を取り合っている）
- Q2: 知りたいこと（相手の気持ち / 復縁の可能性 / 手放すべきか / 新しい出会いの時期）
- Q3: 自由記述

**総合鑑定 → 今後の運勢の流れ**
- Q1: 特に気になる期間（直近1ヶ月 / 3ヶ月先まで / 半年先まで / 1年の流れ）
- Q2: 知りたいこと（仕事 / 恋愛 / 健康 / 金運 / 人間関係 / 転機の時期）
- Q3: 自由記述

**総合鑑定 → 人生の方向性**
- Q1: 現在の状況（岐路に立っている / 漠然とした不安 / 新しいことを始めたい / 今の方向を確認したい）
- Q2: 知りたいこと（適性・才能 / 進むべき方向 / 避けるべきこと / 転機の時期）
- Q3: 自由記述

**総合鑑定 → 今の悩みへのアドバイス**
- Q1: 悩みの領域（仕事 / 恋愛・家庭 / 人間関係 / お金 / 健康 / 漠然とした不安）
- Q2: 今の気持ち（行き詰まっている / 決断できない / 疲れている / 前に進みたい）
- Q3: 自由記述

**運命の転機 → 転機の時期**
- Q1: 気になる分野（仕事 / 恋愛 / 引越し・生活環境 / 人間関係 / 全般）
- Q2: 知りたいこと（いつ動くべきか / 準備すべきこと / チャンスの兆候 / 注意すべきこと）
- Q3: 自由記述

**運命の転機 → 大きな決断の指針**
- Q1: 決断の種類（仕事 / 引越し・移住 / 結婚・離婚 / 独立・起業 / その他）
- Q2: 今の状態（迷っている / ほぼ決めたが不安 / 周囲に反対されている / 期限が迫っている）
- Q3: 自由記述

**運命の転機 → 人生の使命・テーマ**
- Q1: きっかけ（自分の強みを知りたい / 生きがいを探している / 転換期を感じている / スピリチュアルに興味がある）
- Q2: 知りたいこと（魂の目的 / 今世のテーマ / 活かすべき才能 / 乗り越えるべき課題）
- Q3: 自由記述

### ⑥ 受付完了

- 確認画面で入力内容をサマリー表示
- 「鑑定を受け付けました」
- 「○○さん（呼び名）の鑑定結果は数時間後にお届けします」
- バックグラウンドで鑑定生成開始

## PDF設計

### 3ページ構成

#### P1: 導入 + コア鑑定

- ヘッダー（鑑定日・呼び名・星座・ライフパスナンバー）
- 「○○さんの星の配置」— 太陽/月/アセンダント（出生時刻ありの場合）の3要素サマリー
- 今の星の流れ（トランジット概要、具体的な惑星配置に言及）

#### P2: 相談への回答

- 相談内容の引用（Q1〜Q3の回答を要約して記載。「○○さんは今、△△な状況にあり、□□について知りたいとのこと。」）
- タロットリーディング（カード画像 + 解釈、相談内容に直接紐づけた回答）
- 3ヶ月タイムライン（今月→来月→再来月の流れ、具体的な日付入り）

#### P3: 実践ガイド + クロージング

- 好機日・注意日カレンダー（5〜7日ピックアップ、理由付き）
- ラッキーポイント（色・数字・方角 + 具体的なアクション提案）
- 相性の良い星座と活用法（「○○座の人が味方になります」）
- カイからの総括メッセージ（呼び名入り、温かいトーンで締める）

### デザイン

- 背景: ダークネイビー（#0a0a1a → #1a1a2e グラデーション）
- メインテキスト: #d0d0d0、16px、行間 2.1倍
- アクセント: ゴールド #c9a84c
- 重要箇所の太字: #e8d5a3（控えめに、各セクション1〜2箇所）
- タロットカード実画像を含む
- 呼び名を文中で使用（「あなた」ではなく「○○さん」）
- フォント: Noto Serif JP（明朝体系）

## 鑑定生成

### 入力データ（プロンプトに渡す情報）

- 呼び名
- 生年月日 → 星座、ライフパスナンバー、個人年数、個人月数
- 出生時刻（任意）→ アセンダント
- 大項目 + サブメニュー
- Q1 回答（状況）
- Q2 回答（関心事）
- Q3 回答（自由記述、あれば）
- 当日のエフェメリス（惑星配置）
- タロットカード（ランダム抽選）

### プロンプトに含めるべき出力要素

- 具体的な日付（新月・満月・惑星の動きに基づく）
- 相性の良い星座（「○○座の人が味方になる」）
- リスク・注意時期（「○月○日〜○日は△△に注意」）
- ラッキーポイント（色・数字・方角 + 具体的行動）
- 3ヶ月タイムライン
- 好機日・注意日（5〜7日、理由付き）
- 相談内容への直接回答
- 呼び名で呼ぶ

### 遅延配信

- deliver_after = requested_at + 2〜4時間（ランダム）
- cron で15分間隔で pending チェック
- deliver_after を過ぎたものだけ配信

## 新規実装が必要なモジュール

- **数秘術モジュール** — ライフパスナンバー、個人年数、個人月数の計算
- **アセンダント計算**（出生時刻ありの場合）
- **鑑定フロー用 Flex Message** — 紹介カード、基本情報入力、メニュー選択、構造化質問、確認画面、受付完了
- **状態管理** — フローのどのステップにいるかの追跡
- **鑑定リクエスト管理** — KV への保存・取得
- **PDF生成スクリプト** — HTML テンプレート + Playwright で PDF 変換
- **配信スクリプト** — pending リクエストの検出 + LINE Push Message
- **質問セットデータ** — 全12サブメニュー分の Q1/Q2 選択肢

## 決済（後日実装）

②と③の間に Stripe Checkout を挟む設計:

```
② 基本情報入力 → Stripe Checkout → 支払い完了
③ Stripe webhook → メニュー選択カード送信
```
